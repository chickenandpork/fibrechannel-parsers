VirtualWisdom3 -style 2-column format

The NicknameParser interprets a 2-column CSV as though it contains WWPN, Alias For example:

c050761234567890,AIXLeftBig
10000000c9123456,CustServer
50060e801234567f,HDS0123457_CL7F

Additionally, this parser can be used to pull data from any CSV; for example, if your CSV has more than just WWPN/Nickname (which is typical when pulled from another resource), you might have:

BusinessUnit, Server, WWPN, CapEx Code, Location
Billing, BillingBigBox, c050761122334455, 2013A4, Houston
CustServ, Service44, 1000001b32123456, 2013A4, Dallas

You can use system-wide parameters or file-specific parameters to set a different column number (see the jvmopt "wwn", "nickname")

java -D wwn=4 -jar fcparser.jar -N cisco.txt ...
java -D wwn=4 -D nickname=0 -jar fcparser.jar -N cisco.txt ...

better/preferred:

java -jar fcparser.jar -N "cisco.txt;wwn=4" ...
java -jar fcparser.jar -N "cisco.txt;wwn=4;nickname=0" ...

This collector/parser chain is slightly more successful than text-based parsers due to proper interpretation of MS-DOS "I'm the exception" line-ending markers.
