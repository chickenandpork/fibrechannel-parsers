14  * @page NicknameParser VirtualWisdom3 -style 2-column format
15  *
16  * The NicknameParser interprets a 2-column CSV as though it contains WWPN, Alias
17  * For example: @verbatim
18 c050761234567890,AIXLeftBig
19 10000000c9123456,CustServer
20 50060e801234567f,HDS0123457_CL7F
21 @endverbatim
22  *
23  * Additionally, this parser can be used to pull data from any CSV; for example,
24  * if your CSV has more than just WWPN/Nickname (which is typical when pulled
25  * from another resource), you might have: @verbatim
26 BusinessUnit, Server, WWPN, CapEx Code, Location
27 Billing, BillingBigBox, c050761122334455, 2013A4, Houston
28 CustServ, Service44, 1000001b32123456, 2013A4, Dallas
29 @endverbatim
30  *
31  * You can use system-wide parameters or file-specific parameters to set a different column number (see the jvmopt "wwn", "nickname")
32  *
33  * @code java -D wwn=4 -jar fcparser.jar -N cisco.txt ... @endcode
34  * @code java -D wwn=4 -D nickname=0 -jar fcparser.jar -N cisco.txt ... @endcode
35  *
36  * better/preferred:
37  *
38  * @code java -jar fcparser.jar -N "cisco.txt;wwn=4" ... @endcode
39  * @code java -jar fcparser.jar -N "cisco.txt;wwn=4;nickname=0" ... @endcode
40  *
41  * This collector/parser chain is slightly more successful than text-based
42  * parsers due to proper interpretation of MS-DOS "I'm the exception" line-ending markers.
43  */
