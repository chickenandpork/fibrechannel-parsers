24 /**
25  * @page formats Compatible File Formats
26  * @subpage ShowZone2Parser
27  *
28  * @page ShowZone2Parser Cisco "show zone"
29  *
30  * The ShowZone2Parser parses the output of a Cisco SAN switch with the "show zone" command in a more aggressive manner
31  *
32  * This parser was added based on Grzeg Sliwka running into some of the most interesting zones I have seen. I don't think it will handle every situation, as follows:
33  * <ol>
34  * <li>If a zone has two "fcalias name XXX" values, it may be applied to both sets of members</li>
35  * <li>If a zone has a device-alias, the name will not get resolved; this assumes that if the name is known, ever, it'll be somewhere in the file (which will cause a name/wwn mapping to be added)</li>
36  * <li>If a zone has a device-alias, the dereferenced name's WWN will not get added to the zone (yet)</li>
37  * </ol>
38  *
39  * As always, this parser is more successful when the content is treated like
40  * binary: don't edit it. In particular, the use of a putty log to capture output
41  * while querying the switch leads to parser exceptions because the interactive
42  * "you have connected to blah blah system version 1.2.3, the weather is fabulous
43  * and I have 42 pages of boilerplate to spit out before you enter the command for
44  * aliases... hit enter to continue... have some backspaces..." is included. As
45  * are the backspaces.
46  *
47  * When collecting aliases using an ssh command, it's better to execute that command noninteractively:
48  *
49  * In UNIX, Linux, Ubuntu, OSX, or the 6000 other licensees of the USL codebase:
50  *
51  * ssh username@switch44.example.com "show zone" > switch44.shzone
52  *
53  * In windows, using the <a href="http://www.chiark.greenend.org.uk/~sgtatham/putty/download.html">putty</a> tools:
54  *
55  * plink.exe -l username -pw p4ssw0rd switch44.example.com "show zone" > switch44.shzone
56  *
57  * This leads to a better chance to parse 100% of the output with more accuracy,
58  * but I guess following this recommendation is up to the user, and to the engineer
59  * suggesting how he or she should collect the text for parsing.
60  *
61  * @verbatim
62 zone name SANASVR001_FabA vsan 100
63  fcalias name Oracle_123466 vsan 100
64  pwwn 10:00:00:00:c9:12:34:66
65 
66  fcalias name HDS0123457_CL7F vsan 100
67  pwwn 50:06:0e:80:12:34:56:78
68 
69  fcalias name HDS0123457_CL7F vsan 100
70  pwwn 50:06:0e:80:12:34:56:79
71 
72 zone name SANASVR003_FabA vsan 100
73  fcid 0x123456
74  pwwn 50:06:0e:80:12:34:56:79
75  pwwn 10:00:00:00:c9:12:34:67 [Oracle_123467]
76  device-alias SomeUnresolvedHost
77 
78 zone name SANASVR002_FabA vsan 100
79  fcalias name Oracle_123467 vsan 100
80  pwwn 10:00:00:00:c9:12:34:67 @endverbatim
81  *
82  * The resulting content from this example would be the following WWPN/alias pairs:
83  * @verbatim
84 10000000c9123466,"Oracle_123466"
85 10000000c9123467,"Oracle_123467"
86 50060e8012345678,"HDS0123457_CL7F"
87 50060e8012345679,"HDS0123457_CL7F" @endverbatim
88  */
